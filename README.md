# TechSisters Book Club

Members-only reading platform for the TechSisters community.

---

## Local setup

### Prerequisites

- Python 3.11+
- Node.js 20+
- PostgreSQL (via Supabase or local)

### 1. Clone and install

```bash
git clone <repo-url>
cd techsisters-bookclub

python -m venv .venv
source .venv/bin/activate       # Windows: .venv\Scripts\activate
pip install -r requirements.txt

npm install
```

### 2. Configure environment

```bash
cp .env.example .env
# Fill in your values — DATABASE_URL, SECRET_KEY, etc.
```

Generate a secret key:

```bash
python -c "import secrets; print(secrets.token_hex(32))"
```

### 3. Build frontend assets

```bash
npm run build
# Or watch during development:
npm run watch
```

### 4. Run database setup

```bash
# Creates tables (safe to run multiple times)
python -c "
import asyncio
from app.database import create_all_tables
asyncio.run(create_all_tables())
"

# Seed initial data (access code, meeting row, admin user)
python seed.py
```

### 5. Start the server

```bash
python run.py
```

Visit `http://localhost:8000`

## Project structure

app/
├── api/ Route handlers (thin — call services)
├── core/ Config, security, exceptions, middleware
├── models/ SQLAlchemy ORM models
├── repositories/ Async database access layer
├── schemas/ Pydantic request/response validation
├── services/ Business logic
├── static/
│ ├── src/ TypeScript + CSS source files
│ └── dist/ Compiled output (generated — do not edit)
└── templates/
├── pages/ One file per page
├── partials/ Reusable Jinja2 fragments
└── modals/ Modal dialogs included per page
seed.py Seeds access code, meeting, admin user
run.py Dev and production server launcher

## Database migrations

`create_all_tables()` only creates missing tables.
For schema changes in production, use Alembic:

```bash
# Generate a migration after changing models
alembic revision --autogenerate -m "describe the change"

# Apply migrations
alembic upgrade head

# Check current state
alembic current
```

## Deployment (Render)

1. Push to GitHub
2. Create a new Web Service on Render, connect the repo
3. Render detects `render.yaml` automatically
4. Set environment variables in Render dashboard:
   - `DATABASE_URL` — Supabase session mode pooler (port 5432)
   - `HCAPTCHA_SECRET`
   - `RESEND_API_KEY`
   - `ADMIN_EMAIL`
   - `WHATSAPP_GROUP_LINK`
   - `SECRET_KEY` — auto-generated by Render
5. Deploy
6. Run seed script once after first deploy:

```bash
# From your local machine with production DATABASE_URL
ENVIRONMENT=production python seed.py
```
